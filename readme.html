Hooks

Deterministic guardrails

Hooks will get the strongest treatment here: lifecycle points, common recipes, security use cases, and direct links into the working examples.

Why Hooks matter

Instructions guide the model. Hooks execute commands at guaranteed lifecycle points and can block unsafe actions.

What to automate first

Auto-formatting, command blocking, audit logs, and session context injection are the strongest early wins.

Lifecycle points you should remember

SessionStart adds context, PreToolUse blocks or rewrites risky actions, PostToolUse runs validation or formatting, and Stop can force the agent to finish required work.

Best first Hook recipes

Start with a PostToolUse formatter, then a PreToolUse security gate, then SessionStart context injection, then audit logging if you need traceability.

Where people get burned

Hooks run with VS Code's permissions. Validate input, avoid hardcoded secrets, and keep the agent from editing its own hook scripts without manual approval.

VS Code Tasks

Automation outside the model

This section will cover tasks.json as the native runner for builds, tests, problem matchers, background watchers, and Windows shell details.

Task types

Use auto-detected npm, TypeScript, Gulp, and similar tasks when possible; use shell or process tasks when you need full control.

Problem matchers

Attach built-in or custom matchers so task output feeds the Problems panel instead of staying buried in terminal text.

Windows details

On PowerShell, quoting and escaping matter. Prefer command plus args over one large shell string when you can.

How Tasks fit with Copilot

Use Copilot customizations to shape agent behavior. Use tasks.json to run build, test, lint, watch, and validation workflows in the editor or Agents window regardless of model behavior.

What to learn first

Start with task types, groups, and presentation. Then learn dependsOn, runOptions, and problem matchers. Only after that should you spend time on custom multiline matchers or background-watch tasks.

Windows-specific guidance

Prefer a command plus args array. If you must use PowerShell shell strings, be explicit with quoting. For automation-wide behavior, use terminal.integrated.automationProfile.windows or options.shell.
